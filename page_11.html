The Bazel & Rust Ecosystem

Why Bazel?

Unified Polyglot Build Bazel manages the entire monorepo, including C, SystemVerilog, and Rust, ensuring they link and interoperate correctly.

Reproducible Testing Actions run in sandboxes to ensure that build outputs are identical regardless of the environment (Linux, Mac, or CI).

Aggressive Caching Massive build speed improvements by re-running only what has changed since the last git pull.

Rust Usage Examples

opentitantool & opentitanlib The primary interface for interacting with the silicon (bootstrap, reset, I/O) is written in Rust for safety and reliability.

Silicon Validation (SiVal) Host-side test cases are written in Rust to drive functional test vectors while protecting internal silicon state.

rules_rust Integration The project utilizes rules_rust to import dependencies from Cargo.toml into Bazel's optimized build graph.
