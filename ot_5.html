Why Rust?

Memory Safety by Default prevents common classes of critical security bugs

Strong Type System and modern error handling improve reliability

Zero-cost Abstractions matching traditional C/C++ performance

Safer Concurrency for complex multi-threaded host interactions

Chip Firmware / ROM

Why C & ASM?

Early Boot Sovereignty — forms the base of the Trusted Computing Base

Deterministic Behavior and tight control of memory layout

Minimal Runtime Footprint — no allocator/GC or timing jitter

ASM for Reset & Traps — used for reset, traps, and constant-time primitives

Evolution of Firmware: While C is the current standard for ROM, OpenTitan's TockOS ecosystem uses Rust for stable firmware to extend memory safety directly into the embedded silicon layers.
